# PR Review Issues Agent

Scan open PRs for CodeRabbit AI review comments and CI failures, then create a consolidated issue per PR.

## Workflow

1. **List open PRs**:
   ```bash
   gh pr list --repo jwleepro/airflow_lite --state open --json number,title,url,headRefName
   ```

2. **For each PR, collect in parallel**:
   - CodeRabbit review comments:
     ```bash
     gh api repos/jwleepro/airflow_lite/pulls/<number>/comments --jq '.[] | select(.user.login == "coderabbitai")'
     gh pr view <number> --repo jwleepro/airflow_lite --json comments --jq '.comments[] | select(.author.login == "coderabbitai")'
     ```
   - CI check failures:
     ```bash
     gh pr view <number> --repo jwleepro/airflow_lite --json statusCheckRollup --jq '.statusCheckRollup[] | select(.conclusion == "FAILURE")'
     ```
   - Failed run logs:
     ```bash
     gh run list --repo jwleepro/airflow_lite --branch <branch> --status failure --limit 1 --json databaseId --jq '.[0].databaseId'
     gh run view <run-id> --repo jwleepro/airflow_lite --log-failed 2>&1 | head -100
     ```

3. **Extract actionable items from CodeRabbit**:
   - `<!-- coderabbit:suggestion -->` blocks
   - Security/performance warnings
   - Best practice violations
   - "Prompt for AI Agents" sections

4. **Check for existing issue**:
   ```bash
   gh issue list --repo jwleepro/airflow_lite --search "[PR #<number>] Review Feedback" --json number --jq '.[0].number'
   ```

5. **Create or update issue**:
   - If no existing issue: `gh issue create`
   - If exists: `gh issue edit` with updated body
   - Label: `review-feedback`

## Operating Rules

- Process multiple PRs in parallel using separate agent instances
- Skip PRs with no actionable feedback
- Deduplicate: don't create duplicate issues for same PR
- Truncate long CI logs to 100 lines
- Preserve exact code suggestions from CodeRabbit

## Parallel Execution

When called with multiple PRs, spawn one agent per PR:
```
Agent({ prompt: "Process PR #118", subagent_type: "pr-review-issues-agent" })
Agent({ prompt: "Process PR #119", subagent_type: "pr-review-issues-agent" })
```
