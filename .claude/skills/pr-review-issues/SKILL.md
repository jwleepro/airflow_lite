# PR Review Issues Collector

Scan open PRs for CodeRabbit AI review comments and CI failures, then create a consolidated issue per PR.

## Workflow

1. **List open PRs**:
   ```bash
   gh pr list --repo jwleepro/airflow_lite --state open --json number,title,url,headRefName
   ```

2. **For each PR, collect**:
   - CodeRabbit review comments (author: `coderabbitai`)
   - CI check failures (lint, test)
   - Review comments with actionable suggestions

3. **Extract from CodeRabbit comments**:
   - Code suggestions (```suggestion blocks)
   - Security warnings
   - Performance concerns
   - Best practice violations

4. **Extract from CI failures**:
   ```bash
   gh run list --repo jwleepro/airflow_lite --branch <branch> --status failure --limit 1 --json databaseId
   gh run view <run-id> --repo jwleepro/airflow_lite --log-failed
   ```

5. **Check for existing issue**:
   - Search for issue with title pattern: `[PR #<number>] Review Feedback`
   - If exists, update it; if not, create new

6. **Create/Update issue**:
   ```bash
   gh issue create --repo jwleepro/airflow_lite \
     --title "[PR #<number>] Review Feedback: <pr-title>" \
     --body "<formatted-body>" \
     --label "review-feedback"
   ```

```markdown
- [ ] <issue description>

<suggested changes>

### Lint 오류
\`\`\`
<lint errors>
\`\`\`

### Test 실패
\`\`\`
<test failures>
\`\`\`

<extracted prompt if present>

Auto-generated by pr-review-issues skill
```

## Operating Rules

- Skip PRs that already have an up-to-date review issue
- Only create issues for PRs with actual feedback (not empty reviews)
- Label issues with `review-feedback` for easy filtering
- Include checkbox format for trackable action items
- Preserve code suggestion blocks exactly as CodeRabbit provided

## Usage

```
/pr-review-issues           # Process all open PRs
/pr-review-issues 118       # Process specific PR #118
```
